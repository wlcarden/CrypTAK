## Two Parallel Data Paths for Mesh Nodes

Mesh nodes appear in **two places** via **two independent data paths**:

### 1. Map Markers (FTS ‚Üí fn_cot ‚Üí worldmap)
- **Source**: `mesh-relay` converts mesh positions to CoT XML, sends to FTS
- **CoT type** determined by `cot_type` in `nodes.yaml` (e.g., `a-f-G-E-V` for vehicle)
- **Processing**: `fn_cot` in Node-RED receives CoT from FTS TCP, calls `parseCotToMarker()` in `cot-maps.js`
- **SIDC**: Generated by `cotTypeToSIDC()` from the CoT type string
- **Trackers**: Get affiliation/asset-type overrides from flow context (set via popup API)

### 2. Sidebar Panel (MQTT ‚Üí fn_mesh ‚Üí mesh API ‚Üí sidebar JS)
- **Source**: MQTT `msh/+/2/json/#` topics (Meshtastic JSON format)
- **Processing**: `fn_mesh` (Parse Mesh Positions) updates `meshRegistry` flow context
- **API**: `GET /tak-map/api/mesh/nodes` serves registry with SIDC from `nodes.yaml` config
- **SIDC**: Generated by `cotTypeToSIDC()` called in `Serve Mesh Nodes` function
- **Sidebar**: `meshNodeRow()` in `index.html` renders milsymbol icons from API `sidc` field

### Why Two Paths?
- **Map markers** go through FTS so ATAK clients see them too (bidirectional)
- **Sidebar panel** provides rich telemetry (battery, SNR, uptime, topology) not available in CoT

## nodes.yaml (Single Source of Truth)

Located at `firmware/nodes.yaml`. Defines all owned mesh nodes:

```yaml
nodes:
  CrypTAK-BRG01:
    id: "!55c6ddbc"
    profile: bridge
    cot_type: "a-f-G-E-X"    # comms equipment (determines map symbol)
    bridge: true              # explicit bridge flag for sidebar
    tracker: false            # default
  CrypTAK-VHC01:
    id: "!9aa4baf0"
    profile: vehicle
    cot_type: "a-f-G-E-V"    # vehicle equipment
  CrypTAK-TRK01:
    id: "!dce7b97d"
    profile: tracker
    cot_type: "a-f-G-E-V"    # default vehicle (changeable via popup)
    tracker: true             # enables affiliation/asset-type overrides
```

Read by:
- `mesh-relay` ‚Üí determines CoT type sent to FTS
- Node-RED `Mesh Node Whitelist` ‚Üí determines sidebar SIDC + owned/community split

## Tracker Classification (Dynamic)

Trackers can be reclassified in the field via the marker popup:

| Action | API Endpoint | Effect |
|--------|-------------|--------|
| Set affiliation (F/N/U/S/H) | `POST /tak-map/api/tracker/affiliation` | Changes 2525B frame shape + color |
| Set asset type (vehicle/person/cargo/sensor) | `POST /tak-map/api/tracker/asset-type` | Changes function icon within frame |

Both stored in Node-RED flow context (`trackerAffiliations`, `trackerAssetTypes`).
Combined SIDC regenerated via `buildTrackerSIDC(affiliation, assetType)` in `cot-maps.js`.

## NOAA Weather Alerts

- Polled every 5 minutes from `api.weather.gov`
- Rendered as GeoJSON overlay with severity-based colors
- `worldmap` overlay handler auto-replaces on each poll (no manual clear needed)
- Style via `feature.properties.style` with keys: `stroke`, `fill`, `fill-opacity`
